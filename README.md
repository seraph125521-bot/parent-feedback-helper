# 初高中数学课后反馈助手（parent-feedback-helper）

帮初高中数学老师**一键生成发给家长的课后反馈**的轻量 H5 工具。
老师课后填几句课堂观察、作业安排和学生表现，工具会按照老师自己的反馈格式，自动生成可发到家长群的课后反馈。

## 现在是什么状态

- **演示版（MVP v0）**：纯前端 H5，无需后端、无需 API Key，打开即用。
- 反馈内容当前由**初高中数学本地词库 + 老师专属格式模板**生成，用于地推演示和验证需求。
- 老师可以编辑自己的固定格式，例如“时间 / 课次 / 学生 / 学习知识点 / 作业布置 / 老师点评及建议”。
- 已预留接入真实大模型的接口：后续优先替换 `generator.js` 里的 `buildTeacherComment()` 或 `generateFeedback()`。

## 本次已实现的模板能力

### 多页面结构（按使用场景分类）

- 改为底部标签导航的多页面结构，老师按场景切换，不再一个长页面滚到底：
   - **写反馈**：分两步卡片——「这节课的信息」（时间/课次/主题/作业/班级整体）+「这节课的学员」（名单 + 语气）。
   - **我的格式**：单独的发送格式设置页，调好一次长期生效。
   - **生成结果**：作为独立结果页，自带返回 / 重新生成。
- 顶部标题随页面切换，结果页自动隐藏底部导航避免误触。

### 站在老师角度重做的「我的格式」

- 不再让老师面对一段 `{占位符}` 文本，改为**可视化栏目卡片**，老师用日常语言理解：
   - 每个栏目（基本信息 / 学习知识点 / 作业布置 / 老师点评及建议）可**开关**、**上下移动排序**、**改显示小标题**。
   - 顶部可改「开头标题」（如“课后反馈 / 今日课堂小结”）。
   - 底部「实时预览」用示例数据展示家长会收到的真实样子，改一处即时看到效果。
   - 关闭某栏目后编号（一/二/三）自动重排，空内容栏目自动跳过。
- 模板能力升级为**结构化配置**：`templateEngine.js` 新增 `DEFAULT_TEMPLATE_CONFIG`、`SECTION_META`、`renderFromConfig()`、`normalizeConfig()`、`describeSections()`；同时保留旧的 `renderTemplate()` 字符串模板以向后兼容。
- `generator.js` 优先使用结构化配置（`templateConfig`）渲染；未提供时回退旧字符串模板路径。
- 表单内容与格式配置保存到浏览器 localStorage（key 升级为 `pfh_math_sec_state_v2`），刷新后不丢。

## 移动端适配记录

- 底部标签导航固定在底部，适配 `safe-area-inset`（全面屏底部安全区），主内容预留底部留白避免被导航遮挡。
- 「我的格式」栏目卡片在窄屏下把“小标题”输入换行，开关/上下移动按钮在 360px 以下收紧尺寸，保证可点。
- 学员行从平均分布改为响应式网格：宽手机显示“姓名 / 课堂表现 / 删除”，窄手机自动变成“姓名 / 删除”一行、“课堂表现”单独一行。
- 输入框设置 `min-width: 0` 和窄屏断点，避免长占位文案把学员名单区域挤变形。
- 360px 以下屏幕会收紧页面左右边距和输入框内边距，优先保证表单可填写。

## 怎么运行

不需要任何依赖，任选一种：

```bash
# 方式一：Python 自带服务器（推荐）
python -m http.server 8000
# 然后浏览器打开 http://localhost:8000

# 方式二：直接双击 index.html 也能用（复制功能在 http 下更稳）
```

手机上演示：把这些静态文件传到任意静态托管（如 Vercel、Netlify、GitHub Pages、或国内对象存储），得到一个网址，私信发给老师即可。

## 默认反馈格式（结构化栏目）

老师在「我的格式」页里看到的是四个可视化栏目，按推荐顺序：

| 栏目 | 默认小标题 | 内容 |
| --- | --- | --- |
| 基本信息 | 基本信息 | 时间、课次、学生姓名 |
| 学习知识点 | 学习知识点 | 这节课讲了什么 |
| 作业布置 | 作业布置 | 课后要完成的作业 |
| 老师点评及建议 | 老师点评及宝贵建议 | 亮点 + 待加强 + 学习建议 |

生成示例（默认全开）：

```text
课后反馈

一. 基本信息
时间：2026年6月15日
课次：第8次课
学生：张晨

二. 学习知识点
一次函数图像与性质

三. 作业布置
完成专题试卷第 1-12 题

四. 老师点评及宝贵建议
……（自动生成的点评）
```

每个栏目都可开关、排序、改小标题；老师无需理解 `{占位符}`。

## 接入真实大模型（验证通过后再做）

1. 选一个国内大模型（DeepSeek / 通义 / 豆包），拿到 API Key。
2. 加一个极简后端（云函数/Serverless）转发请求，**不要把 Key 放前端**。
3. 在 `generator.js` 里优先替换 `buildTeacherComment()`：
   - 让大模型生成 `{teacherComment}`；
   - 继续使用 `templateEngine.js` 套老师自己的模板。
4. 如果希望大模型一次性输出完整反馈，可以使用 `buildFeedbackPrompt(input)` 生成包含老师模板的完整提示词。
